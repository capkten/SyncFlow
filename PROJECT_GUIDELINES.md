# 项目开发规范

为了确保项目开发的高效与一致性，本项目遵循以下规范：

## 1. 语言规范
- **沟通语言**：所有项目相关的讨论、文档及交流默认使用**中文**。
- **代码注释**：代码中的所有注释、文档字符串（Docstrings）必须使用**中文**编写。
- **变量命名**：变量、函数、类名等标识符使用英文（应遵循各语言通用的代码风格，如 Python 的 PEP8，JavaScript 的 ESLint 规范等），但在必要时需配合中文注释进行说明。
- **输出信息**：用户界面（UI）或命令行输出（CLI）的提示信息，原则上使用**中文**。

## 2. 项目管理
- **版本控制**：统一使用 **Git** 进行代码版本管理。
- **提交信息 (Commit Message)**：提交信息应清晰明了，建议使用中文描述具体的修改内容（例如：“修复登录页面的崩溃问题” 或 “新增用户注册功能”）。
- **分支管理**：建议使用 `main` 或 `master` 作为主分支，开发新功能时创建新的特性分支。

## 3. 文件及目录结构
- 文件名建议使用英文，以保持跨平台兼容性。
- 目录结构应保持清晰，逻辑分明。

---
*本规范文件作为项目的最高指导原则，后续开发需严格遵守。*
